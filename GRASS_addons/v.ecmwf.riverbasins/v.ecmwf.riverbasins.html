DESCRIPTION

v.ecmwf.riverbasins updates the list of all the partner institute IDs intersecting with each river basin.

The aim of this procedure is that for each basin in the hydrological model we want to know: 1) what partner regions regions lie within this basin. The basins become an attribute called 'MOUids_all' (MOU = Memorandum of Understanding) which records a list all the partner institute IDs which lie within each river basin.

The input partner regions store the separate polygon areas for each individual partner institute, rather than the dissolved partner regions that we have been working with previously.

The input river basins are given in a separate datasource/layer.

The workflow is as follows:

  1. Intersect the river basins with the individual (un-dissolved) partner regions shapefile to identify partner regions which intersect with each river basin.
  2. Loop through each of the river basins.
  3. Extract the partner ID for each of these partner regions which intersect the corresponding river basin and assign them to the 'MOUids_all' attribute of the river basin.

EXAMPLE

Update river basin attributes with new partner regions. 
v.ecmwf.riverbasins partner_regions=ESoWC/MOUids_EFAS_LAEA_edited_v4.shp \
		    basins=ESoWC/mainbasins_EU_EFAS_LAEA_ETRS89_v2_clean.shp \
		    output=ESoWC/mainbasins_new.shp
